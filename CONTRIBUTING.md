# Contributing

Thanks for helping improve this educational BMSSP repository.

## Goals

Contributions should make the project:

- clearer,
- more accurate,
- more visual,
- easier to teach,
- or better connected to the original literature.

## Ground Rules

Please do not:

- claim this repository invents BMSSP,
- add unsupported benchmark claims,
- present conjectures as proven results,
- or simplify the algorithm in a way that changes its meaning without warning.

Please do:

- cite the academic source when adding theorem-level statements,
- mark simplified explanations as simplified,
- keep diagrams readable,
- and prefer small focused changes.

## Documentation Style

Use:

- clear headings,
- short paragraphs,
- Markdown tables,
- Mermaid diagrams,
- and warning or note blocks where they help.

## Suggested Contributions

- More worked examples.
- Visual animations.
- A paper notation guide.
- A small reference implementation clearly labeled as educational.
- Comparison notes with Dijkstra, Bellman-Ford, A*, and multi-source Dijkstra.

## Review Checklist

Before opening a pull request, check:

- [ ] The change does not claim new research results.
- [ ] The wording distinguishes explanation from theorem.
- [ ] Mermaid diagrams render on GitHub.
- [ ] Tables are readable on mobile.
- [ ] Any complexity statement is sourced or phrased cautiously.
